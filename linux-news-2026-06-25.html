🐧 Linux 每日动态

2026年6月25日 · 图形·显示·计算 专题

📋 今日概览

本期聚焦 Linux 生态中图形、显示与计算方向的最新进展。Linux 7.2 内核合并窗口已开启，涵盖大量 GPU 驱动改进；AMD 持续推进 HDMI 2.1 合规与显示精度优化；开源 NVIDIA NVK 驱动实现 DLSS 支持，标志着开源图形栈的重要突破；Valve 发布光线追踪检查器工具助力驱动优化；Xfce 首个 Wayland 合成器预览版发布，桌面显示系统持续演进。

1
内核 GPU 显示
Linux 7.2 内核合并窗口开启 — 图形子系统大规模更新
Linux 7.2 合并窗口于 6 月 14 日随 7.1 版本发布后正式开启，截至 6 月 18 日已有超过 7000 个非合并补丁合入主线。本次更新涵盖 AMDGPU、Intel i915、Nouveau 等主流 GPU 驱动的显著改进，包括 DRM 子系统优化、显示电源管理增强、以及对新 GPU 架构的前期支持。同时，sched_ext 调度框架持续优化，有望提升 GPU 异构计算场景下的任务调度效率。
值得关注：Linux 7.2 内核将直接影响下一代 Linux 发行版的图形与计算能力。DRM 子系统的改进、AMD/Intel/NVIDIA 驱动的同步推进，意味着 Linux 在 AI 计算、游戏图形、专业显示等领域的竞争力将持续提升。建议 GPU 驱动开发者、Linux 桌面用户密切关注此版本的演进。
2
AMDGPU 显示 驱动
AMD 提交 Gamma 2.4/2.6 曲线支持补丁 — 专业显示色彩精度提升
6 月 24 日，AMD 向 Linux 内核邮件列表提交了新补丁，为 AMDGPU 开源驱动新增 Gamma 2.4 和 Gamma 2.6 曲线支持。Gamma 曲线是专业色彩管理的关键参数，直接影响显示器的色彩准确性和 HDR 表现。此前 AMDGPU 驱动仅支持标准 sRGB Gamma 2.2，新补丁将大幅改善专业图形工作、视频后期等领域在 Linux 下的色彩体验。配合此前已开始的 HDMI 2.1 合规性测试准备，AMD 在 Linux 显示领域的投入持续加码。
值得关注：对于从事专业图形、视频制作、色彩关键型工作的 Linux 用户而言，Gamma 曲线支持的完善是长期以来的痛点。此补丁将缩小 Linux 与 Windows/macOS 在专业显示领域的差距。同时，HDMI 2.1 合规准备的推进也预示着更高带宽、可变刷新率（VRR）等先进显示特性即将在 Linux 上全面到位。
3
NVIDIA 计算 开源
开源 NVK Vulkan 驱动首次支持 DLSS — Linux 游戏图形重大突破
6 月 19 日，Mesa 项目的开源 NVIDIA Vulkan 驱动 NVK 宣布初步支持 DLSS（深度学习超级采样）技术。DLSS 是 NVIDIA 基于 AI 的超分辨率技术，能在保持画质的同时显著提升游戏帧率。此前 DLSS 仅能在 NVIDIA 闭源驱动下使用，NVK 的开源实现意味着使用开源驱动栈的 Linux 用户也能享受这一先进图形技术。此进展得益于 NVK 驱动对 NVIDIA 专有扩展的逆向工程与实现，是开源图形生态的重要里程碑。
值得关注：NVK 支持 DLSS 是开源 Linux 图形栈的历史性突破，将显著提升 Linux 游戏体验，缩小与 Windows 的差距。对于希望使用开源驱动、同时不牺牲图形性能的玩家而言，这标志着 NVK 已具备进入实用阶段的潜力。此外，这也对 NVIDIA 闭源驱动形成竞争压力，可能加速其开源化的进程。

GPU计算 工具 驱动优化
Valve 发布光线追踪检查器 RTI — 助力 Linux GPU 驱动优化
6 月 22 日，Valve 宣布创建光线追踪检查器（Ray-Tracing Inspector, RTI）工具，用于帮助图形开发者进一步优化 Linux 下的 GPU 驱动。RTI 支持 AMD RADV、Intel ANV、NVIDIA NVK 等主流开源 Vulkan 驱动，能够深入分析光线追踪工作负载的性能瓶颈，提供详细的流水线分析数据。该工具是 Valve 持续投入 Linux 图形生态的最新举措，此前 Valve 还推动了早期 AMD GCN GPU 的 Recovery 机制改进，以及修复了存在多年的 Radeon 显示 Bug。
值得关注：Valve 作为 Linux 游戏生态的重要推动者，其发布的 RTI 工具将显著降低 GPU 驱动优化的门槛，帮助开源驱动追赶闭源驱动的光线追踪性能。对于 Linux 游戏开发者、GPU 驱动开发者而言，这是一款不可多得的调试利器。同时也预示着 Linux 上的光线追踪游戏体验即将迎来新的提升。
5
Wayland 桌面 显示服务器
Xfce Wayland 合成器首个预览版发布 — 传统桌面迈向现代显示架构
6 月 22 日，Xfce 桌面环境宣布其 Wayland 合成器 xfwl4 发布首个预览/Alpha 版本。经过近 6 个月的开发，xfwl4 已实现基本的功能集，目标是在 Wayland 下提供与 X11 会话尽可能接近的使用体验。Wayland 是现代Linux 显示架构的主流方向，具备更简单的代码架构、更好的安全性、以及原生支持高 DPI 和混合刷新率显示器等优势。Xfce 作为轻量级传统桌面，其 Wayland 迁移标志着 Linux 桌面生态整体向现代显示架构的转型进入新阶段。
值得关注：Xfce Wayland 合成器的发布，意味着几乎所有主流 Linux 桌面环境（GNOME、KDE、Xfce、LXQt 等）均已启动或完成 Wayland 迁移。对于使用轻量级桌面的用户、以及企业 Linux 部署场景而言，这是重要的里程碑。Wayland 的全面普及将解决 Linux 长期存在的混合显卡、HiDPI 缩放、安全隔离等显示问题，是 Linux 桌面体验进阶的关键一步。
