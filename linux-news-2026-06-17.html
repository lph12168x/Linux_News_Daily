Linux 7.1 稳定版正式发布 内核 GPU NPU
6月14日·Linus Torvalds·kernel.org

Linus Torvalds 于 6 月 14 日正式发布 Linux 7.1 稳定版内核，距 7.0 仅 9 周，累计超 10,000+ 次提交。

与图形/显示/计算相关的核心更新：

  • GPU 驱动增强 — AMDGPU、Intel Xe、NVIDIA Nova 三个驱动栈均获得大量修复与优化，GPU 驱动补丁占 RC7 首位
  • NPU 加速器驱动 — 继续增强 AMD Ryzen AI NPU 监控与调度能力（详见第 2 条）
  • 全新 NTFS 内核驱动 — 由 Namjae Jeon 开发，替代旧驱动，提供完整 NTFS 读写支持（a.k.a.「NTFS 的复兴」）
  • 文件系统 + BPF 扩展 — Btrfs RAID 可靠性提升，bcachefs 逼近生产就绪，新 BPF 程序类型持续模糊内核与用户空间边界
  • 调度器增强 — cgroup 感知调度合并，对容器和数据库场景意义重大

同步发布 7.0.12 稳定更新分支，安全修复持续跟进。GNU Linux-libre 7.1-gnu 亦于次日发布。

Linux 7.1 是 7.x 系列首个功能更新，奠定了未来 9 周内所有上层发行版（Ubuntu、Fedora、Arch）的内核基线。对 GPU/NPU 驱动的增强直接影响桌面渲染、AI 推理和高性能计算三大工作负载。NTFS 驱动的重写则消除了跨平台数据交换的长期痛点。

Intel Compute Runtime 宣布支持 Nova Lake NPU，引入实验性 "LEO" NPU 计算
6月15日·Intel / Phoronix

Intel 开源 Compute Runtime（面向 OpenCL 和 oneAPI Level Zero）发布 26.22.38646.4 版本：

  • 将 Nova Lake Xe3P（第六代 NPU 架构）支持状态正式升级为 "早期支持"（Early Support）
  • 同时引入代号 "LEO" 的实验性特性，标志着 Intel 正加速将 NPU 作为一等公民融入其统一计算堆栈
  • Nova Lake 预计 2026 年底出货，集成第六代 NPU，端侧 AI 算力预期大幅跃升
  • 开源 Compute Runtime 的提前支持意味着开发者在硬件上市前即可进行软件适配与性能预热
NPU 正从"移动端的特色"变为"桌面/服务器的标配"。Intel 在硬件未上市前就通过开源驱动提前布局，这是非常明确的信号：端侧 AI 推理将成为下一代 CPU 平台的核心卖点。对从事 NPU/加速器开发的工程师而言，这意味着 Linux 下的统一编程模型（OpenCL / oneAPI）将覆盖 CPU、GPU 和 NPU 三类算力。

Weston 16 Alpha 发布：HDR 改进 + Vulkan 渲染器修复 + Wine Wayland 分数缩放 显示 GPU
6月15日·freedesktop.org / Phoronix

Wayland 参考合成器 Weston 16.0 Alpha 1 发布，多项图形与显示栈改进：

  • HDR 改进 — 作为 Wayland 色彩管理协议（color-management-v1）落地后的首个参考实现更新，Weston 16 Alpha 进一步打磨 HDR 元数据处理与色调映射管线
  • Vulkan 渲染器修复 — 修复了 Weston 内嵌 Vulkan 后端在特定场景下的渲染异常，提升稳定性

同时期，Wine Wayland 驱动程序合并了分数缩放（Fractional Scaling）支持，从根本上解决了 HiDPI 显示器上 Windows 应用模糊/过小的问题。Wine 11.11（6月12日）及 Wine-Staging 11.11（289 个实验补丁）同步发布。

HDR + 分数缩放是 Linux 桌面从"开发者可用"迈向"普通用户可用"的两大关键门槛。Weston 的 HDR 改进为 GNOME/KDE 合成器提供了参考路径；Wine Wayland 的分数缩放则直接提升 Linux 下运行 Windows 生产力软件（如 Office、Adobe）的可用性。三者叠加形成了图形/显示栈的"三位一体"进步。

Intel 开源项目 "Performance Skills"：AI 驱动的 Linux 性能分析 计算 AI
6月15日·Intel / Phoronix

Intel 发布全新开源项目 "Intel Performance Skills"：

  • 利用 AI 代理（Agent） 自动分析 Linux 上的 CPU 性能瓶颈，生成优化建议
  • 目标是将 perf、top-down 微架构分析等工具链的经验知识编码为 AI 可消费的"技能"，降低性能调优门槛
  • 开源并面向社区，鼓励第三方贡献新的分析技能

互补新闻：Linux 7.2 中 /proc/filesystems 读取性能提升 444%（该文件被 SELinux 库高频调用）。仅移动两行代码，EXT4/XFS 的 IOmap 就为 IOPS 带来 +5% 的提升——经典的内核微优化案例。

AI 辅助性能分析工具的出现标志着 Linux 性能工程正在从"专家手工调优"向"AI 自动化诊断"转型。对于服务器端 GPU/NPU 计算场景（如 RDMA + GPU Direct），CPU 侧的性能瓶颈往往是整体吞吐的隐形天花板；Intel Performance Skills 这类工具可以直接帮助定位这些隐藏瓶颈。

Linux 7.2 合并缓存感知调度（Cache-Aware Scheduling） 计算 内核
6月15日·LKML / Phoronix

Linux 7.2 合并窗口（6 月 15 日开启）首批合并的重大特性之一——缓存感知调度器：

  • 针对具有多个最后级缓存（LLC / Last-Level Cache）的现代 CPU 设计，能感知任务在不同 LLC 域间的迁移开销
  • 对 AMD EPYC（多 CCD）、Intel Xeon（多 tile/chiplet）等 NUMA 拓扑下的大规模并行计算负载有显著收益
  • 同时合并了匿名管道写入性能优化（提升 Shell 管道及进程间通信速度）和容器卸载延迟的大幅降低

其他 7.2 合并窗口亮点：Rust Zerocopy 库（40,000+ 行 Rust 代码）引入内核以消除更多 unsafe 代码；Btrfs 默认启用 Large Folios + 引入 Huge Folios；XFS Zone Allocator 脱实验；移除 40 年历史的 Hercules ISA 显卡驱动。

缓存感知调度是现代多核 CPU 上高性能计算的基础设施。在 RDMA + GPU Direct 等高吞吐场景中，内存访问局部性（NUMA locality）与缓存命中率是决定端到端延迟的关键因素。7.2 的调度器改进意味着上层框架无需手动绑核/绑 NUMA 就能获得更好的近端访问性能，对分布式存储、数据库和大模型推理等场景是直接利好。
