Valve 发布 RTI 光线追踪检查器：为 Linux GPU 驱动优化提供专业级调试工具 GPU 计算
6月22日·Phoronix / freedesktop GitLab

Valve 的开源 Linux 图形团队向 Mesa 26.1 合入了 Ray-Tracing Inspector（RTI），一款专为分析和优化 Vulkan 光线追踪性能而设计的 GUI 调试工具。

核心要点：

  • 开发者定位 — RTI 用于检查 RT 加速结构（BVH）和光线追踪分发调用，非终端用户工具，面向驱动开发者和性能工程师
  • 技术栈 — 基于 ImGUI + Vulkan + SDL3 构建，由 Valve 工程师 Konstantin Seurer 主导开发
  • 当前聚焦 RADV — 初期仅支持 Radeon RADV Vulkan 驱动，通过 RADV_DEBUG=rti 导出 BVH/射线历史数据至 /tmp 后加载分析
  • 可扩展架构 — 设计上预留了多后端接口，未来可扩展至其他 GPU 厂商（NVK、ANV 等）
  • 编译方式 — 通过 -D tools=rti Meson 构建选项启用
Linux 开源图形栈一直缺乏专业级光追调试工具，开发者通常依赖 GPU 厂商闭源工具链（如 NVIDIA Nsight）或打印日志。RTI 填补了开源领域的空白：① 对 RADV 驱动的光追优化将因可视化而显著提速 — 直接影响 Steam Deck / SteamOS 上的光追游戏体验；② Valve 持续投资开源图形工具链（此前已有 Fossilize、VKD3D-Proton），RTI 是光追时代的标志性基础设施；③ 可扩展架构意味着 NVK 等下一代开源驱动的光追优化也能受益。

Steam Machine 正式发布：$1049–$1428，AMD Zen 4 + RDNA3，4K/60FPS Linux 游戏主机 GPU 显示
6月22日·Phoronix / SteamPowered

Valve 正式公布 Steam Machine 定价与规格，这台运行 SteamOS Linux 的客厅游戏 PC 将 Steam Deck 的开放生态延伸至桌面级性能：

  • 硬件规格 — 半定制 AMD Zen 4 APU（6C/12T）、RDNA3 28 CU 集显、16GB DDR5 系统内存 + 8GB GDDR6 显存
  • 性能目标 — 宣称 6 倍于 Steam Deck 性能，支持 4K@60FPS + FSR
  • 定价档位 — 512GB 版 $1,049（含手柄 $1,128）、2TB 版 $1,349、顶配含手柄 $1,428
  • 发售机制 — 6 月 25 日开启一次性随机预购队列，应对产能受限
  • 软件生态 — SteamOS 基于 Arch Linux，Mesa RADV 驱动默认就绪，VKD3D-Proton + DXVK 处理 Windows 游戏兼容
Steam Machine 是 Valve 自 2015 年首代 Steam Machines 失败后，时隔 11 年再次冲击 Linux 游戏主机市场。相较于首次尝试，这次的底气来自：① Steam Deck 已验证"Proton + Linux"的可行性，SteamOS 装机量已达数千万；② RDNA3 架构在 Mesa RADV 驱动下的表现已非常成熟，不存在 2015 年 NVIDIA 闭源驱动 + Big Picture 的散架局面；③ 对 GPU/显示领域而言，这意味着一台"出厂即支持 Vulkan 1.3、光追、HDR"的 Linux 游戏盒，将倒逼更多游戏厂商在开发阶段就考虑 Linux 兼容性与性能调优。

Mesa 26.2 合入 Vulkan Present Timing：X11/XWayland 帧时序支持落地 显示 GPU
6月21日·Phoronix / freedesktop GitLab (MR!39551)

经过四个月代码审查，Mesa Vulkan WSI 代码正式合入 VK_EXT_present_timing 扩展的 X11/XWayland 后端支持，预计随 8 月 Mesa 26.2 稳定版发布：

  • 解决的问题 — 让应用获取 前一帧的实际呈现时间戳 反馈，实现精准帧节奏控制和 卡顿消除，同时降低输入延迟
  • 此前局限 — Wayland 原生路径已有 Present Timing 支持，但大量 Steam Play 游戏仍运行在 X11/XWayland 上，一直无法受益
  • 贡献者 — Valve 工程师 Hans-Kristian Arntzen（VKD3D-Proton 作者）主导，经历 4 个月代码审查后合并
  • 实现范围 — 覆盖基础 X11 后端 + XWayland 路径，确保无论用户选择 X11 还是 Wayland 桌面，均能获得帧时序改进

VK_EXT_present_timing 于 2025 年 11 月随 Vulkan 1.4.335 正式发布，旨在替代不精确的 vkAcquireNextImageKHR 时序模型。

帧时序问题是 Linux 游戏体验中的长期短板：① 缺乏精确的 present 时间反馈导致 微卡顿 难以消除，尤其在高刷新率显示器上更明显；② Wayland 原生游戏已受益一年，而占比更大的 XWayland 游戏群体一直"被搁置" — 此次合入终于消除了这一分裂；③ 无论对开发者还是用户，这意味着 Linux 上 Vulkan 游戏的流畅度将在 Mesa 26.2 发布后获得统一提升，不再依赖于"你用什么桌面协议"。

Linux 7.2 开始为 NVIDIA "Blackwell-Next" 做准备：下一代 GPU VFIO/CXL 路径亮相 计算 内核
6月21日·Phoronix / LKML (VFIO pull request)

Linux 7.2 合并窗口的 VFIO 子系统补丁中首次出现了 "Blackwell-Next" 代码引用，NVIDIA 软件工程师正在为下一代 GPU 的 Linux 内核支持提前铺路：

  • 涉及路径 — NVGrace-GPU VFIO 驱动中 CXL（Compute Express Link） DVSEC 处理逻辑的适配，以应对 Blackwell-Next 相比现有 Blackwell 的变化
  • 代号谜团 — "Blackwell-Next"具体指向 Blackwell Ultra (B300)、Rubin 架构，还是全新的产品线，补丁中未明确披露
  • CXL 意义 — CXL 是 NVIDIA Grace-Hopper 超算平台上 CPU-GPU 互联的关键通道，线路变更意味下一代 GPU 的内存一致性模型或拓扑可能发生调整
  • 补丁时机 — 2026 年 6 月初编写，6 月 15 日随 VFIO 拉取请求提交，目标 7.2 主线合入 — 比硬件发布提前 6–12 个月

NVIDIA 近年已显著改善上游 Linux 贡献节奏，从早期 Grace-Hopper 到 Blackwell 再到如今的 Blackwell-Next，内核前哨补丁的提前期越来越短，反映其对 Linux 生态的重视。

对关注高性能计算和 GPU 的开发者而言：① 内核中的 "Blackwell-Next" 引用是 下一代 NVIDIA 数据中心 GPU 的早期信号 — CXL 路径变更可能预示新的互联拓扑或内存架构；② NVIDIA 持续在上游合入 CXL/PCIe/VFIO 硬件使能代码，表明 Grace-Hopper 超算平台的 Linux 主线路线图仍在推进，并非只依赖 OEM 分支；③ 对于希望在下一代 GPU 上抢先适配的 HPC 和 AI 基础设施团队，这是开始评估兼容性的时间窗口。

Miracle-WM 0.10 发布：Mir 基 Wayland 合成器引入插件化渲染管线，目标年底 v1.0 显示
6月22日·Phoronix / GitHub (miracle-wm-org)

Canonical 工程师 Matthew Kosarek 发布 Miracle-WM 0.10，这是基于 Mir 显示服务器的 Wayland 合成器项目的重大功能更新：

  • 插件化渲染管线 — 插件现可为每个窗口设置自定义着色器管线（如焦点模糊效果），也可为整个输出设置管线（如夜间模式/护眼）
  • 协议覆盖扩大 — 新增支持 ext_data_control_manager_v1、ext_image_copy_capture_manager_v1、ext_output_image_capture_source_manager_v1、ext_foreign_toplevel_list_v1、zwp_input_method_v1 等协议
  • 生态就绪 — Grimshot、wl-copy、xdg-desktop-portal-wlr 现已在 Miracle-WM 上正常运行
  • GTK4 集成 — 新增 GTK4 配置错误报告客户端和调试叠加层
  • v1.0 路线图 — 开发者明确目标在 2026 年底前发布 1.0 正式版
Wayland 合成器生态长期由 wlroots 主导（Sway、Hyprland、River 等），Miracle-WM 选择 Mir 作为基础是一种差异化路径：① Mir 本身由 Canonical 维护，拥有独立的输入/渲染管线实现，与 wlroots 形成良性竞争；② 插件化渲染管线设计更接近现代合成器架构（类似 KWin Effects），允许社区开发丰富的视觉效果而不修改核心代码；③ 如果 Miracle-WM 按计划在年底达到 1.0，Wayland 阵营将首次拥有一个由大厂背书的、非 wlroots 的成熟平铺/浮动混合合成器。
