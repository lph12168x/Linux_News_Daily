Valve 发布 Proton 11.0-1 稳定版：基于 Wine 11.0 重构，新增多款游戏支持
GPU 计算
Valve 今日发布 Proton 11.0-1 稳定版，这是驱动 Steam Play、Steam Deck 与新款 Steam Machine 的关键组件。该版本将 Proton 重新基于上游 Wine 11.0 重构，自 4 月起历经多轮测试版后首发稳定版。相比 Proton 10.0，它新增支持一批 Windows 游戏：包括 Universe Generator: The Golden Sword、DCS World、生化危机（1996/1998）、恐龙危机 1/2、X-Plane 12、Deadly Premonition 等首次可在 Proton 下运行；并修复 Rockstar Launcher 弹窗渲染、诸多 EA 游戏在 EA Desktop 更新后无法运行、Steam Overlay 失效等问题。组件方面同步升级至 Xalia 0.4.9、FEX 2605（ARM64EC）、VKD3D 1.19 Git、DXVK 2.7 Git、DXVK-NVAPI 0.9.1、Wine Mono 11.0、VKD3D-Proton 20260410。
关注理由：Proton 是 Linux 游戏生态的核心引擎，11.0-1 基于 Wine 11.0 重构并带来大量新游戏支持与渲染修复，DXVK/VKD3D 同步升级直接改善 AMD/NVIDIA 显卡上的游戏兼容性与帧表现。

AMD 着手清除 AMDGPU 内核驱动中全部 BUG() 调用（30 补丁系列）
GPU 内核
AMDGPU 内核驱动维护者 Alex Deucher 今日发布一组 30 个补丁，着手清除该 Linux 内核图形驱动中全部 BUG() 宏的使用。Linux 的 BUG() 宏用于在遇到不可恢复状况时主动使内核崩溃并转储调试信息，但调用它会引发完整内核 panic，也可能被攻击者用于制造拒绝服务。Deucher 逐一审查 AMDGPU 中的 BUG() 调用，将无需主动崩溃且可恢复的场景改为仅告警（warn）或输出错误但让驱动安全继续运行。该 30 补丁系列已发布至 AMD 图形邮件列表评审。
关注理由：大量 BUG() 调用使 AMDGPU 在遇到异常时直接拖垮整个系统，改为优雅降级后用户设备在遇到驱动问题时能继续运行而非内核崩溃，显著提升 AMD 显卡 Linux 系统的稳定性与可用性。

NVIDIA 将下一代 Rosa CPU 的 Rigel 核心初始支持上游化至 GCC
计算
在 NVIDIA 公布下一代 Rosa CPU 的 Rigel 核心细节后数分钟，针对该核心的初始 GCC 支持即被上游化至 GNU Compiler Collection 代码库。Rosa CPU 采用 Rigel 核心，是采用 Olympus 核心的 Vera CPU 的继任者；NVIDIA 确认 Rigel 为基于 Armv9.2-A 的自研核心。编译器工程师 Dhruv Chawla 将初始 CPU target 合入 GCC Git，新增 -mcpu=rigel 以瞄准下一代 Rosa CPU，首批提交复用 Olympus 的 ISA 能力（part ID 0x11，Olympus 为 0x10），暂未包含专门调优。该支持赶在明年的 GCC 17 稳定版之前落地。
关注理由：NVIDIA 自研 Arm CPU 生态进入工具链上游，意味着 Linux 发行版与开发者能在 GCC 17 中直接为 Rigel 生成优化代码，对 agentic AI 时代的高性能计算与超算（Vera Rubin 路线）意义重大。

NVIDIA 公布下一代 Rosa CPU 细节：Rigel 核心（Armv9.2-A）
计算
NVIDIA 在一篇主打 Vera CPU 单线程性能的博客中，确认了下一代 Rosa CPU 及其 Rigel 核心的部分细节。Rosa 是 Vera（Olympus 核心）的继任者，沿用相同硅片面积。NVIDIA 表示 Rigel 是其下一代 Arm v9.2 CPU 核心，相比 Olympus 提供更高的每核性能，关键改进包括更好的指令派发、更大的 L2 缓存与更高效的内存处理。Rosa 将延续 Vera 作为 agentic AI 处理器的定位，在更高单核性能、更大 L2 与更高效内存处理之间取得平衡，后续将与 AMD EPYC Venice、Intel Xeon Diamond Rapids 等服务器 CPU 正面竞争。
关注理由：NVIDIA 的 CPU 路线图（Vera→Rosa）直接服务于 AI 超级计算，Rigel 核心在单核性能与内存效率上的改进将影响下一代 Grace/Blackwell 平台的 Linux 体验与 HPC 应用表现。

GPU 显示

关注理由：作为 R610 分支的月度更新，610.43.03 主要面向 Linux 7.1 稳定支持的稳定性打磨，NVIDIA 闭源驱动对最新内核的及时适配是 Linux 桌面与工作站用户持续关注的重点。

Razer 认证首款 Linux 笔记本：Blade 18（RTX 5090 + 4K 屏）
显示 GPU
Razer 在 2026 年首次为其笔记本申请 Ubuntu Linux 认证，机型为 Razer Blade 18 RZ09-0582。该机搭载 Intel Core Ultra 9 290HX Plus（24 核：8P+16E，5.5GHz，Arrow Lake）与 NVIDIA GeForce RTX 5090（24GB GDDR7）独显，配 3840×2400 显示屏。在 Ubuntu 26.04 LTS（Linux 7.0 内核）上，得益于成熟的 Arrow Lake 支持与状态良好的 RTX 5090 Blackwell 驱动，整机运行顺畅，ACPI 平台配置等也正常工作。Razer 表示该机型尚不被社区维护的 OpenRazer 支持，但 RGB 控制之外核心体验良好；目前认证仍在进行中。顶配（128GB RAM）售价达 $6999，低配约 $3499。
关注理由：游戏硬件大厂 Razer 首次推进 Linux 认证，标志着高端游戏本对 Ubuntu 的官方支持起步；Arrow Lake + RTX 5090 Blackwell 在 Linux 上的顺畅体验，也印证了 Intel/NVIDIA 图形栈与最新内核的成熟度。
