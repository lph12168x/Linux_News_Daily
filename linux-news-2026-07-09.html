AMD 发布 ZenDNN 6.0：大幅加速 Ryzen / EPYC CPU 上的 AI 推理
计算
AMD 今日发布 ZenDNN 6.0，这是面向 AMD Zen 处理器（从 Ryzen 到 EPYC）的开源深度神经网络库重大更新，用于加速本地 AI 推理。ZenDNN 是 AMD 对标 Intel / UXL Foundation oneDNN 的方案，可通过插件与 TensorFlow、PyTorch 协作。值得注意的是，本次发布早于官方 AMD Zen 6 处理器登场。6.0 版本在 Low Overhead API（LowOHA）、扩展的 MoE 分组 GEMM 与 FP16 算子覆盖、以及后处理（post-op）与权重缓存方面做了大量改进，下载与细节见 GitHub 发布页。
关注理由：ZenDNN 是 AMD CPU 上 AI 推理的关键加速库，6.0 在 MoE / FP16 与权重缓存上的改进直接利好本地大模型推理与生成式 AI 负载，对 Linux 上的 CPU 计算生态意义重大，也为 Zen 6 平台预热。

Linux Mint 下一代 Cinnamon 中 Wayland 不再标记为「实验性」
显示
Linux Mint 发布 6 月开发小结，其中最引人注目的是 Cinnamon 的 Wayland 支持已准备「毕业」。从下一代 Cinnamon 版本起，Wayland 支持将不再被视为「实验性」，且 X11 与 Wayland 都将获得完整支持。开发团队表示 Wayland 体验已「几乎与 X11 持平」。期间完成了新窗口 / 菜单的正确映射、焦点窃取防护、更多 Wayland 崩溃场景修复、多显示器支持改进、硬件加速增强、完整 HiDPI 与窗口进度等。此外 Muffin 现在会对所有 Clutter 对象的坐标与尺寸做取整。
关注理由：作为最流行的桌面发行版之一，Linux Mint 将 Wayland 从实验性提升为完整支持，标志着 Wayland 在主流桌面的最终落地又迈进一步，对 Linux 显示栈的普及具有风向标意义。

XWayland 24.1.13 发布：修复 X.Org 代码库中两处新安全漏洞
显示 GPU
X.Org Server 代码库今日披露两处新安全漏洞，XWayland 同样受影响。Trend Micro Zero Day Initiative 在 Claude Opus 4.6 协助下持续发现新的 X.Org 漏洞。Red Hat 的 Peter Hutterer 公布 CVE-2026-55999（GLAMOR 字体代码中的缓冲区溢出）与 CVE-2026-56000（GLX 代码路径中的释放后使用 UAF）。XWayland 24.1.13 已包含这些修复及若干其他修复；X.Org Server 21.1.24 也已发布相应安全更新。
关注理由：GLAMOR 依赖 GPU 加速的 2D 渲染，GLX 则是 X11 下的 OpenGL 扩展，这两处漏洞直接关系到运行 X11 程序（通过 XWayland）时显卡相关代码路径的安全，Linux 桌面用户应及时更新。

Intel 本周归档 Quantum Intrinsics 等多个开源 AI / 计算项目
计算
Intel 本周正式归档了多个不再维护的开源项目。继 2 月归档 Quantum Passes 后，本周又归档了 Quantum Intrinsics——一个基于 LLVM/Clang 的 C++ 前端，用于 Intel Quantum Compiler。此外，尽管 AI 热潮正盛，Intel 仍在持续收缩部分 AI 开源项目：此前已归档 BigDL 时间序列工具包与 IPEX-LLM，本周 OAP-MLLIB 成为最新被归档的项目。OAP-MLLIB 提供优化的 Spark MLlib 机器学习加速包，底层依赖已移交 UXL 基金会的 oneDAL，并面向 Intel CPU 与 GPU 能力。同时被放弃的还有面向 Intel Xeon 服务器压缩算法优化的 Intel FastGo（Go 语言包）。
关注理由：这反映了 Intel 在 AI 与 HPC 开源生态上的战略收缩，oneDAL / oneCCL 已移交 UXL 基金会；对依赖 OAP-MLLIB 等在 Intel CPU / GPU 上做 Spark ML 加速的用户而言，需关注迁移路径。

Intel Core Ultra 7 270K Plus 单 / 双通道内存性能基准（含代码编译）
计算
应付费读者请求，Phoronix 在 Intel Core Ultra 7 270K Plus（Arrow Lake）桌面上实测了单通道与双通道 DDR5 内存的性能差异。测试在 Ubuntu 26.04（Linux 7.0 内核）上进行，仅更换内存配置（单条 Crucial Pro DDR5-6400 32GB vs 双条 2×16GB DDR5-6400），运行大量 Linux 性能基准，读者特别关注代码编译性能。在当前内存价格高企背景下，该测试为「先买单条、待降价再补双通道」的装机思路提供了量化参考。
关注理由：内存带宽直接影响 CPU 计算与编译密集型任务，对 Linux 开发者与 HPC 工作站的硬件选型具有实际指导意义，也侧面印证 Arrow Lake 平台在最新 Ubuntu / Linux 7.0 上的成熟度。
