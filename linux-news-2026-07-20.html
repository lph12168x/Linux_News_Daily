Linux 7.2-rc4 发布：修复节奏进入「新常态」
内核
Linux 7.2 的第四个每周候选版本（rc4）现已可供测试。Linus Torvalds 在公告中形容本周「看起来相当正常」，只是这个「正常」已变成大量散落各处的修复并存的「新常态」——rc4 并不算小。统计上仍以大量一两行的细小改动居多，较大的修复出现在 can/bcm（syzbot 与 sashiko 修复）和 selftests 上；Linus 还特别提到 diff 里有一处「较大的 AMD GPU 噪声」，但解释称主要是代码搬运与新增注释，并非以往那种庞大的头文件删减。他同时指出，本周不少修复仍是由 AI / LLM 辅助发现的。Linux 7.2 预计将在 8 月以稳定版形式登场。
关注理由：rc4 是 7.2 周期中段的关键检查点。本期要点不在新特性，而在两方面：一是 AI/LLM 辅助发现修复已成为内核常态化的「新常态」节奏；二是 AMDGPU 仍是内核中改动最密集、最易制造 diff「噪声」的子系统之一，侧面印证 GPU 驱动主线活跃度。对跟踪 7.2 特性集（8 月稳定、将作为 Ubuntu 26.10 等发行版内核基础）的用户，rc4 是判断稳定性走向的合适节点。

FFmpeg 落地最新 AVX-512 优化：像素格式转换快 1.372 倍
计算
资深 FFmpeg 开发者 Niklas Haas 扩展了 FFmpeg 的 libswscale，使其利用 AVX-512 的「按字节向量置换」（Vector Permute Byte, VPERMB）指令来加速视频缩放与像素格式转换。借助 VPERMB，代码可在跨通道（cross-lane）层面实现更高效的字节置换，从而改善非 lane 对齐场景的表现。实测下来，在支持 AVX-512 的处理器上，RGB24 → RGBA 的 1920×1080 转换速度提升了 1.372 倍。这对 AMD Zen 4 及更新 CPU、以及支持 AVX-512 的 Intel Xeon 等处理器是好消息。
关注理由：像素格式转换是视频管线中最频繁的基础操作之一，FFmpeg 这种手写 AVX-512 汇编优化能直接加速现代 x86 Linux 上的转码、缩放与串流负载。1.372 倍提升看似 modest，却落在「每一帧都会跑到」的热路径上，累积收益可观；同时它也延续了业界对 CPU 计算侧（而非仅 GPU）在媒体栈持续投入的趋势，对跑在服务器/工作站 CPU 上的视频处理用户尤其有意义。

联想 Legion Y700 Gen4（骁龙 8 Elite）现身 Linux 支持补丁
显示 内核
搭载高通骁龙 8 Elite 的 8.8 英寸 Android 平板联想 Legion Y700 Gen4（最高 16GB RAM、1TB 存储、1904×3040 IPS 显示屏）如今出现了脱离 Android、启用主线 Linux 支持的初步补丁。开源贡献者 Pengyu Luo 今天发出了建立该平板 Device Tree 的初版补丁，可让按键、USB、frame-buffer 显示等基础功能正常工作；借助下游（downstream）补丁还能进一步驱动 GPU 与触摸屏。不过 WiFi、蓝牙、摄像头、麦克风、传感器与振动马达等仍仅在下游代码中可用，离「主线可用」还有距离，但有望随时间完善。
关注理由：又一台主流 Android 设备向主线 Linux 迈进：初版补丁已点亮 frame-buffer 显示，配合下游补丁更能跑通 Adreno（骁龙 8 Elite 内置）GPU 与触摸屏，与本项目关注的「显示 / 图形」方向直接相关。它体现了在骁龙 8 Elite 这类现代 ARM 平台上运行纯主线 Linux 的持续努力，对移动 Linux、ARM 图形栈与平板 DIY 用户都具有跟踪价值——尽管当前仍是「能用基础功能」的早期阶段。
