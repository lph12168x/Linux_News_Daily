GNOME Mutter 实现 GPU 重置恢复功能 — GSoC 2026 项目取得实质进展
显示 GPU GNOME
Phoronix · 2026-07-03 · GNOME / GSoC

目前，在 Linux 下如果遇到 GPU 重置（GPU reset），GNOME 会话会直接崩溃退出。 这一长期存在的用户体验问题，正由 Google Summer of Code 2026 项目着手解决—— 学生开发者 Toluwaleke Ogundipe 正在为 GNOME Mutter 合成器实现健壮的 GPU 重置恢复处理。

该项目基于 OpenGL 的 GL_EXT_robustness 扩展及类似基础设施， 目前已进展到 Mutter 合成器可以在 GPU 重置后保持会话稳定的阶段： 窗口能正确更新、输入设备恢复响应、会话不再崩溃或冻结。

待解决的项包括：帧缓冲区尚未自动重建、桌面背景渲染异常、以及部分残留的 OpenGL 错误。 这些项目仍在工作中，完成后将提交上游合并请求。 详细进展可参阅 开发者博客文章。

关注理由： GPU 重置恢复是 Linux 桌面稳定性的重要短板，尤其在使用 AMD 或 Intel 开源驱动时， 偶发的 GPU 挂起会导致整个桌面会话丢失。Mutter 的这一改进将显著提升 GNOME 桌面的 健壮性，是 Linux 桌面用户体验的关键底层改进。

Vulkan 1.4.356 发布：新增 OCP Microscaling MX 格式扩展，加速机器学习推理
计算 GPU Vulkan
Phoronix · 2026-07-03 · Khronos / Vulkan

Vulkan 1.4.356 于今日发布，最大亮点是新增了 VK_EXT_shader_ocp_microscaling_types 扩展， 为 Vulkan 带来对 Open Compute Project (OCP) Microscaling Formats (MX) 的支持， 以提升机器学习工作负载的性能。

OCP Microscaling 格式包括 MXFP4、MXFP6、MXFP8 和 MXINT8， 旨在通过低精度数据格式提高 AI 训练和推理的计算吞吐量与降低内存带宽需求。 该扩展支持 OCP MX 格式规范中定义的浮点类型，但暂不提供对 microscaled tensors 或直接对 microscaled 数据进行操作的支持。

该扩展由 NVIDIA、Samsung、Imagination 和 Arm 共同贡献， 过去数月间持续开发。行业对新型 AI 数据格式的需求推动了这一扩展的落地， 使得 Vulkan 生态在机器学习推理场景下的竞争力进一步增强。

关注理由： Microscaling 格式是 AI 推理领域的重要技术方向（已获 NVIDIA Blackwell、AMD CDNA 4 等新一代架构支持）， Vulkan 生态对此的标准化支持，将使得跨平台（包括 Linux 上的 Vulkan 计算场景）的 AI 推理应用能更高效地利用低精度计算能力，是 GPU 计算生态的重要进展。

Intel 为 Linux 7.3 提交 Nova Lake 显示驱动补丁：启用 CMTG 与 DC3CO
显示 GPU 内核
Phoronix · 2026-07-03 · Intel / Linux DRM

Intel 于 7 月 2 日向 Linux 内核社区提交了首批面向 Linux 7.3 的 drm-intel-next 新特性补丁，重点继续围绕下一代 Nova Lake 处理器的 Xe3P 集成显卡/显示支持展开。

本次提交的两个关键新特性：

  • CMTG（Common Mode Timing Generator）支持 — Nova Lake 显示模块的 CMTG 功能已接通。CMTG 首次出现在 Lunar Lake 硬件上， 可用于多显示器之间的时序同步（如 eDP 笔记本屏幕与外部 DisplayPort 显示器之间的时序协调）。
  • DC3CO（Display C3 with Clock Off）支持 — 一种先进的显示省电特性，在 PSR2 空闲期间允许显示引擎的电源域完全断电， 同时保持显示图像，可显著降低平台功耗，对电池供电设备尤为关键。

Linux 7.3 合并窗口预计将于 8 月下旬开启，Intel 未来数周还将提交更多 Nova Lake 图形驱动优化补丁。Nova Lake 预计 2026 年底或 2027 年初发布， 集成 Coyote Cove 性能核与 Arctic Wolf 能效核，以及 Xe3P 集成显卡。

关注理由： Nova Lake 是 Intel 下一代重要客户端处理器平台，其 Linux 图形驱动的早期上游化 体现了 Intel 对 Linux 桌面/工作站支持的持续投入。CMTG 和 DC3CO 的启用 尤其利好多显示器笔记本用户和移动平台的电池续航表现。

NVIDIA 将 Vera Rubin VR-NVL 超级计算机 BMC 设备树上游化，支持 OpenBMC
GPU 计算 数据中心
Phoronix · 2026-07-03 · NVIDIA / Linux 内核

NVIDIA 近期在 Linux 内核邮件列表上发布了其 Vera Rubin VR-NVL 服务器平台 的基板管理控制器（BMC）Device Tree 补丁，这是 NVIDIA 为支持开源 OpenBMC 软件而上游化的最新举措。

该补丁系列包含一份 901 行的 Device Tree 文件，描述 VR-NVL BMC 的硬件配置。 NVIDIA VR-NVL 是面向 Vera Rubin NVL72 机架级超级计算机的平台， 其 BMC 基于 ASpeed AST2600 控制器。

与此同时，NVIDIA 也向 U-Boot/OpenBMC 社区提交了配套补丁。 OpenBMC 是基于 Linux 的开源基板管理控制器固件栈，近年来在超大规模数据中心中 因其安全性和透明度优势而获得越来越广泛的采用（AMD 和 Intel 服务器平台也已大量采用）。

关注理由： NVIDIA 主动将 Vera Rubin 新一代 AI 超级计算机平台的 BMC 支持上游化至开源社区， 标志着 NVIDIA 在数据中心开源生态中的参与度持续提升。对运行 Linux 的 AI 训练集群管理员而言，OpenBMC 支持意味着更透明的硬件管理能力和更高的安全性。
