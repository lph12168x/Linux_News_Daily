NVIDIA 为 Nova 开源驱动开发新 TLV 固件二进制格式
📅 2026-07-01 🔗 Phoronix
NVIDIA 工程师正在为正在开发中的开源 Nova 内核驱动设计一种新的 GPU 固件镜像二进制格式——TLV（Type-Length-Value）格式。TLV 是嵌入式系统和通信协议中广泛使用的数据结构，其自描述特性使得基于 Rust 语言编写的 Nova 驱动代码能更简洁地解析固件内容。Nova 是 NVIDIA 全新开发的 Rust 内核驱动，旨在为 Hopper、Blackwell 及后续 GPU 架构提供一流的开源支持，与现有的 Nouveau 驱动并行发展。
💡 关注理由 Nova 驱动代表着 NVIDIA 对 Linux 开源生态的战略投入。TLV 固件格式的选择体现了 Rust-first 的设计理念，有望降低固件解析的复杂度，加速 Nova 驱动的开发迭代。

COSMIC Epoch 1.2 修复 Intel 新显卡闪烁问题
📅 2026-06-30 🔗 Phoronix
System76 发布了基于 Rust 开发的 COSMIC Epoch 1.2 桌面环境。本次更新的重要修复针对 Intel 新一代显卡的显示问题：自 Meteor Lake 以来，用户在使用集成显卡与独立显卡（AMD/NVIDIA）混合配置的笔记本时，会遭遇画面闪烁。问题延续到了 Arrow Lake 和 Lunar Lake。COSMIC 合成器通过在 Intel iGPU 与独立 GPU 之间复制数据时不再尝试 DMA 拷贝来规避该问题。此外，还修复了工作区概览打开时的闪烁、新增 AVIF 图片支持等。
💡 关注理由 Intel 新平台（Meteor Lake / Arrow Lake / Lunar Lake）上的图形体验直接影响大量笔记本用户。COSMIC 的修复为 Rust 桌面生态在混合 GPU 场景下的可靠性树立了标杆。

Valve 工程师修复 AMD GCN 1.0/1.1 GPU 驱动 VM 故障
📅 2026-07-01 🔗 Phoronix
Valve 的 Linux 图形驱动工程师 Timur Kristóf 持续关注 AMD 老一代 GPU 的开源驱动支持。最新修复针对 GCN 1.0 (GFX6) 和 GCN 1.1 (GFX7) GPU 的硬件/固件缺陷：EVENT_WRITE_EOP 无法等待 L2 缓存写回，导致栅栏（fence）在 L2 仍为脏状态时就发出信号，引发 VM 故障和 GPU 挂起。修复方案是在每次提交后等待着色器完成并刷新 L2 缓存。此外还回退了之前针对 Hawaii GPU 的 RADV 临时 workaround（内核端已有正式修复），避免了该 workaround 带来的性能退化。
💡 关注理由 Valve 对老旧硬件的持续投入体现了 Linux 游戏生态的长尾价值——即使是近十年前的 GCN 1.0/1.1 显卡，仍能获得关键的稳定性修复，受益于 Steam Deck 生态的反哺效应。

Linux 7.3 引入 DRM 色彩格式属性，AMDGPU 率先支持
📅 2026-06-28 🔗 Phoronix
虽然 Linux 7.2 合并窗口刚刚关闭，Linux 7.3 周期的首个重要特性已经浮现：DRM 子系统将引入新的"色彩格式"属性，允许用户空间请求显示驱动以特定色彩格式输出。支持的初始值包括 AUTO、RGB、YUV 4:4:4、YUV 4:2:2、YUV 4:2:0。该特性经过 15 轮修订，最早可追溯到 2021 年的早期探索。AMDGPU 内核驱动在首批拉取请求中率先获得支持，Intel 和 Rockchip 驱动的补丁也已提交。Wayland 的 Weston 合成器已准备好利用该属性。
💡 关注理由 DRM 色彩格式属性是 Linux 显示栈色彩管理的重要里程碑，YUV 格式的原生支持对视频播放、HDR 内容呈现和广色域显示器具有实际价值。AMD 在此领域继续保持领先。

Intel 砍掉 AMX-TF32：Diamond Rapids 的 AI 矩阵加速未出道即夭折
📅 2026-06-30 🔗 Phoronix
Intel 在最新版的 ISA 编程参考手册中移除了 AMX-TF32 指令扩展，同时删除的还有 User-Timer Events 和 Interrupts。AMX-TF32 原计划为 Advanced Matrix Extensions (AMX) 引入 NVIDIA TensorFloat-32 (TF32) 格式的原生支持——TF32 兼具 FP32 的数值范围和 FP16 的计算性能，是 AI/矩阵运算的关键加速格式。该特性自 2024 年被记录为 Diamond Rapids 的 ISA 能力，如今在处理器计划于 2027 年发布前被突然移除。Intel 工程师已同步从 GCC 编译器中剥离 AMX-TF32 支持，并需要向 GCC 15/16 回溯移除补丁。这与去年被砍掉的 AMX-TRANSPOSE 遭遇了相同命运。
💡 关注理由 AMX-TF32 的取消可能反映 Intel 在数据中心 AI 加速路线图上的调整。对于依赖 AMX 进行 AI 推理加速的 Linux 计算生态而言，这意味着 Diamond Rapids 在矩阵计算方面的差异化优势减弱。

Linux 7.3 修复 PCIe Gen5 NVMe 小 I/O 瓶颈，随机读取提升 14%
📅 2026-07-01 🔗 Phoronix
字节跳动工程师 Fengnan Chang 在 Linux 内核中发现了一个影响 PCIe Gen5 NVMe SSD 小 I/O 性能的"显著瓶颈"。在进行 4K 随机读取时，CPU 将大量时间消耗在 IOmap 框架的内存分配和状态机处理上。解决方案是在 IOmap 的直接 I/O 路径中为小型请求（I/O 大小 ≤ inode 块大小）引入一条简化路径。测试结果显示，4K 随机读取从 1.92M IOPS 跃升至 2.19M IOPS；在 EXT4/XFS + io_uring + 高队列深度场景下，性能提升可达 10%。该补丁已合入 VFS 子系统的 vfs-7.3.iomap 分支。
💡 关注理由 PCIe Gen5 NVMe 正快速普及至工作站和数据中心。这个看似"微小"的内核优化在高性能计算、数据库和 AI 训练数据加载等 I/O 密集型场景能带来可观的吞吐量收益。
