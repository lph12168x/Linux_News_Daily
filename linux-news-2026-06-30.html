Wine 11.12 发布：Wayland 驱动支持分数缩放
📅 2026-06-29 🔗 Phoronix
Wine 11.12 正式发布，为 Wayland 驱动带来了期待已久的分数缩放支持（fractional scaling），用户在不同显示器上可以使用不同的缩放比例，极大提升了高分辨率显示屏上的 UI 体验。此外，该版本还修复了主显示器位置、调试队列命名等问题，并捆绑了 FFmpeg 的 libswresample 和 libswscale 库，更新了 Mono 11.2 引擎。共修复了 27 个已知 bug，涵盖 Corel 产品安装、图形问题、Microsoft Office 2007 等应用。
💡 关注理由 Wine 的 Wayland 原生支持正在快速成熟，分数缩放是高分辨率显示环境下的关键功能，对于 Linux 桌面用户运行 Windows 应用体验提升显著。

RADV 驱动默认启用 Vulkan 描述符堆支持
📅 2026-06-29 🔗 Phoronix
随着 Mesa 26.2 稳定版预计在 8 月发布，Radeon RADV Vulkan 驱动现已默认启用 VK_EXT_descriptor_heap 描述符堆支持。此前在 Mesa 26.1 中该功能已引入，但需要设置 RADV_EXPERIMENTAL=heap 环境变量才能启用。Valve Linux 图形团队的 Samuel Pitoiset 在确认无已知问题后，决定默认开启该功能。描述符堆扩展允许对描述符和存储描述符的内存进行显式管理，相较 VK_EXT_descriptor_buffer 提供了更可移植的保证和更可预测的性能特征。
💡 关注理由 Vulkan 描述符堆是新一代 GPU 资源管理模式，对游戏和计算性能都有积极影响，RADV 默认启用标志着该技术已足够成熟。

ZLUDA v6 发布：AMD GPU 上 PhysX 运行良好，但失去商业资助
📅 2026-06-29 🔗 Phoronix
开源 CUDA 兼容项目 ZLUDA v6 正式发布，该项目曾秘密获得 AMD 资助开发，后在 2024 年底转向多 GPU CUDA 支持并专注于 AI。遗憾的是，商业资助再次被切断，项目维护者 Andrzej Janik 表示 ZLUDA 将回归"周末项目"状态。尽管如此，v6 版本带来了令人兴奋的进展：PhysX 现已在 AMD GPU 上良好运行，对老游戏的物理效果加速非常有意义。此外，v6 还新增纹理支持，并大幅改进了 Windows 支持（除 Linux 外）。
💡 关注理由 ZLUDA 是 AMD GPU 用户摆脱 NVIDIA CUDA 生态锁定的重要希望，PhysX 支持的达成对游戏用户尤其有价值，但资助不稳定是隐忧。

Linux 7.2-rc1 发布：AMDGPU HDMI 2.1 FRL 等功能合并
📅 2026-06-28 🔗 Phoronix
Linux 7.2 的第一个候选版本（rc1）如约发布，标志着为期两周的合并窗口关闭。Linus Torvalds 表示"事情看起来相当正常"。Linux 7.2 合并了许多新特性，包括Cache Aware Scheduling、Intel USB4STREAM、AMD ISP4 驱动、AMDGPU HDMI 2.1 FRL 支持初始工作等。本次发布包含超过 4300 万行代码，其中约三分之一是 AMD GPU 寄存器定义头文件更新。Linux 7.2 稳定版预计在约八周后的性能测试后发布，将为 Fedora 45 和 Ubuntu 26.10 等发行版提供内核。
💡 关注理由 Linux 7.2 内核带来了多项图形和显示相关改进，AMDGPU HDMI 2.1 FRL 支持对于高清音视频用户非常重要，标志着内核图形栈的持续进化。

Nourish：基于 Vulkan 的新 Wayland 合成器，支持无限工作区
📅 2026-06-27 🔗 Phoronix
一个新的 Wayland 合成器 Nourish 亮相，其独特卖点是提供"无限"缩放和平移功能，实质上创建了一个无限工作区。Nourish 使用 Vulkan API 进行渲染（可回退到 OpenGL ES 以支持旧硬件），设计理念是"不让用户受限于屏幕大小"。该项目已在开源 AMD、Intel GPU 驱动以及官方 NVIDIA 驱动上测试。Nourish 利用 y5（一个用 Rust 编写的 Wayland 合成器框架）进行底层开发，并使用了 Smithay、WGPU、Iced 等项目的代码。目前 Fedora Linux 已有二进制包可用。
💡 关注理由 Nourish 代表了 Wayland 合成器的新方向，Vulkan 渲染和无限工作区概念对创意工作者和高效能用户非常有吸引力，是 Linux 桌面创新的有益尝试。

DXVK 3.0 发布：DXBC-SPIRV 着色器编译，默认启用描述符堆
📅 2026-06-25 🔗 Phoronix
DXVK 3.0 正式发布，这是用于 Wine 和 Valve Steam Play (Proton) 的 Direct3D 8/9/10/11 到 Vulkan 转译层的重要功能更新。主要变化包括：采用 DXBC-SPIRV 进行着色器编译，替代旧版着色器转译代码；默认使用 Vulkan 描述符堆（VK_EXT_descriptor_heap）；大量 Direct3D 改进、共享资源增强、帧率限制器支持等。DXBC-SPIRV 是基于 SSA 的 D3D SM 5.1+ 编译器，生成的代码更紧凑，着色器编译完全卸载到工作线程，有助于缩短游戏加载时间。DXVK 3.0 现在需要 Vulkan 1.4 驱动级别支持。
💡 关注理由 DXVK 是 Linux 游戏体验的关键组件，3.0 版本的着色器编译改进和描述符堆支持将显著提升游戏性能和兼容性。
